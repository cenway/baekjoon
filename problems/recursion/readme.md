재귀는 절차지향적 사고 금지
귀납적 사고 방식
ㄴ이건 헷갈릴 때 마다 바킹독 예시 보기

func(1)을 호출하면 ~가 나온다
func(k)를 호출하면 ~~~가 "나온다면" func(k+1)을 호출하면 k+1~~~~가 나온다를 증명

특정 입력에 대해서는 그냥 종료되어야 함
그리고 모든 입력이 해당 입력으로 수렴되어야 함

피보나치 재귀가 비효율적인 상황 을 기억하자
물론 이미 호출된 값을 저장해서 시간 복잡도를 확 낮출 수 있음


기본적으로 자기를 다시 호출해 문제를 작게 나누고 해결하면 재귀로 보니,
머지 소트처럼 분할해 내려가고 그 결과를 바탕으로 다시 올라오면서 완성하는 것도,
퀵 소트처럼 분할해 내려가면서 완성하고 끝나는 것도 다 재귀로 봄

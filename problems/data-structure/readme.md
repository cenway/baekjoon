## 리스트(container, 반복자로 내부 확인 가)

중간 삽입, 삭제가 필요할 때 좋으나 투 포인터로 대체되는 경우도 종종 존재

배열로 자주 대체됨

## 스택(container adapter, 내부 확인 불가)

아무래도 방향이 정해지면 좋음

뭘 넣어야 하는지 모르겠으면 트리의 dfs처럼 생각해보기

구조체나 pair를 넣는 경우도 자주 있음(인덱스 같은 다른 정보를 함께 남기기 위해서)

## 덱(container, 반복자로 내부 확인 가 ←STL 기준)

*순수한 자료구조 정의 상 안 되는 걸로 아는데 STL은 가능

벡터와 다른 점은 벡터는 배열처럼 연속적인 주소 공간을 가져서 크기가 커지면 맞는 블럭에 재할당해줘야 함

그러나 덱은 리스트처럼 불연속적인 주소 공간을 가져 크기 증가 상수 시이 보장됨

ㄴ중간 삽입 삭제는 list에 비해 성능이 떨어진다고 알고 있음

인접 리스트는 정점이 많고 간선이 적은 경우 유용
인접 행렬로 표현하기엔 공간이 부족할 때
또, 연결된 모든 정점을 확인해야 하는 경우에도 인접 리스트가 나음

인접 리스트는 그냥 벡터로 구현
각 정점(인덱스)와 연결된 정점을 넣어주기
근데 문제는 간선이 뒤지게 많으면 그냥 인접 행렬 쓴 거랑
공간복잡도가 큰 차이 없음. 탐색만 느려지고

시간복잡도를 고려할 때,
bfs의 경우 리스트는 O(V+E), 행렬은 O(V^2)
리스트의 경우 간선이 꽤 큰 비중을 차리할 수 있음

"비재귀 DFS는 순회를 잘 수행하지만 엄밀히 말해 우리가 관념적으로 생각하는 DFS와 방문 순서가 다릅니다. 그래서 단순히 Flood Fill 내지는 순회를 하는 것이 아니라 DFS의 고유한 성질을 사용해 문제를 해결해야 하는 상황일 경우에는 앞에서 소개한 비재귀 코드를 이용하면 안됩니다."
